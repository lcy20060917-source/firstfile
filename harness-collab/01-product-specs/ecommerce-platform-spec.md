## 功能背景

构建一个完整的电商购物平台，提供从商品浏览到下单购买的全流程电商体验。系统采用前后端分离架构，后端基于 Spring Boot 提供 RESTful API，前端基于 Vue 3 构建 SPA 应用。

**当前痛点**：
- 需要一个轻量级、可快速部署的电商平台原型
- 初期使用 SQLite 降低部署门槛，后续可平滑切换至 MySQL
- 前端需要现代化、响应式的用户界面

**预期收益**：
- 完整的电商购物闭环（浏览 → 搜索 → 加购 → 下单 → 订单管理）
- Token 认证保障 API 安全
- 支持 SQLite/MySQL 自由切换，满足不同阶段的部署需求

---

## 目标

**主要目标**：
1. 实现商品展示、搜索、购物车、下单、订单管理等核心电商功能
2. 基于 JWT Token 实现用户认证与授权
3. 支持 SQLite 和 MySQL 数据库，通过配置即可切换
4. 前后端完全分离，通过 RESTful API 通信
5. 建立完整的拦截器体系，记录请求信息

**成功指标**：
- 核心业务流程可用：商品浏览 → 加入购物车 → 下单 → 查看订单
- API 响应时间：正常负载下 ≤ 500ms
- 支持 50+ 并发用户
- 单元测试覆盖率 ≥ 80%

## 用户故事

### 故事 1：用户注册与登录

**As a** 平台用户  
**I want** 注册账号并登录系统  
**So that** 我能使用需要身份认证的功能（购物车、下单等）

#### 验收标准

- **Given** 用户未注册  
  **When** 用户填写用户名、密码、邮箱并提交注册  
  **Then** 系统创建账号并返回成功信息

- **Given** 用户已注册  
  **When** 用户输入正确的用户名和密码  
  **Then** 系统返回 JWT Token，前端存储并用于后续请求

- **Given** 用户已登录  
  **When** Token 过期  
  **Then** 系统返回 401，前端引导用户重新登录

#### 异常场景

- **Given** 用户名已存在  
  **When** 用户注册  
  **Then** 系统返回 409 错误，提示用户名已被占用

- **Given** 用户输入错误密码  
  **When** 用户登录  
  **Then** 系统返回 401 错误，提示用户名或密码错误

---

### 故事 2：商品浏览与搜索

**As a** 平台用户  
**I want** 浏览商品列表并搜索感兴趣的商品  
**So that** 我能找到想要购买的商品

#### 验收标准

- **Given** 系统中有商品数据  
  **When** 用户访问首页  
  **Then** 系统展示商品列表，支持分页

- **Given** 用户在商品列表页  
  **When** 用户输入关键词进行搜索  
  **Then** 系统返回匹配的商品（按名称模糊搜索）

- **Given** 用户在商品列表页  
  **When** 用户点击某个商品  
  **Then** 系统展示该商品的详细信息

- **Given** 用户在商品详情页  
  **When** 页面加载完成  
  **Then** 显示商品名称、描述、价格、库存、图片等信息

---

### 故事 3：购物车管理

**As a** 已登录用户  
**I want** 将商品加入购物车并管理购物车内容  
**So that** 我能暂存想购买的商品，稍后统一下单

#### 验收标准

- **Given** 用户已登录且商品有库存  
  **When** 用户点击"加入购物车"  
  **Then** 商品添加到购物车，更新购物车数量

- **Given** 用户购物车中有商品  
  **When** 用户查看购物车  
  **Then** 展示商品列表、单价、数量、小计金额

- **Given** 用户购物车中有商品  
  **When** 用户修改商品数量或删除商品  
  **Then** 购物车实时更新

#### 异常场景

- **Given** 用户未登录  
  **When** 用户点击"加入购物车"  
  **Then** 系统提示需要登录，跳转登录页

---

### 故事 4：下单与订单管理

**As a** 已登录用户  
**I want** 将购物车中的商品生成订单并查看订单状态  
**So that** 我能完成购买流程并追踪订单

#### 验收标准

- **Given** 用户购物车中有商品且已登录  
  **When** 用户填写收货信息并提交订单  
  **Then** 系统生成订单，清空购物车，扣减库存

- **Given** 用户已下单  
  **When** 用户查看订单列表  
  **Then** 展示所有订单（订单号、金额、状态、时间）

- **Given** 用户已下单  
  **When** 用户点击某个订单  
  **Then** 展示订单详情（商品列表、收货信息、金额明细）

#### 异常场景

- **Given** 商品库存不足  
  **When** 用户提交订单  
  **Then** 系统返回错误，提示库存不足

- **Given** 购物车为空  
  **When** 用户提交订单  
  **Then** 系统返回错误，提示购物车为空

## 非功能性需求

### 性能要求
- **响应时间**：正常负载下 API 响应时间 ≤ 500ms
- **并发量**：支持 50 并发用户同时操作
- **数据量**：支持万级商品数据的查询性能

### 安全要求
- **认证**：所有写操作接口需要 Bearer Token 认证
- **授权**：用户只能操作自己的购物车和订单
- **数据保护**：用户密码使用 BCrypt 加密存储，Token 使用 HMAC-SHA256 签名
- **输入验证**：所有用户输入使用 Hibernate Validation 校验

### 兼容性要求
- **浏览器兼容性**：支持 Chrome 90+、Firefox 88+、Edge 90+
- **数据库兼容**：支持 SQLite 3.x 和 MySQL 8.0+

### 可用性要求
- **接口一致性**：所有 API 使用统一的响应格式 `{ code, message, data }`
- **错误处理**：统一的全局异常处理，返回标准化错误信息
